# Tracking objects using Optical Flow and SIFT Recovery

### Цель работы
Целью данной работы является реализация и анализ гибридного алгоритма трекинга объекта на видео. В отличие от стандартных методов, данная реализация сочетает в себе скорость оптического потока (Lucas-Kanade) и надежность поиска по ключевым точкам (SIFT) для восстановления трекинга при его потере.

### 1. Теоретическая база
Для решения задачи используются два дополняющих друг друга метода:

1.  **Метод Лукаса-Канаде (Optical Flow):** Дифференциальный локальный метод вычисления оптического потока. Он предполагает, что яркость объекта не меняется существенно между кадрами, а движение соседних пикселей согласовано. В коде используется реализация `cv2.calcOpticalFlowPyrLK` с использованием пирамид изображений, что позволяет отслеживать более быстрые перемещения.
2.  **SIFT (Scale-Invariant Feature Transform):** Алгоритм поиска ключевых точек, инвариантных к масштабу и повороту. Он используется в качестве «страховки»: если оптический поток теряет объект, SIFT-дескрипторы позволяют заново локализовать его в любой части кадра.
3.  **Гомография и RANSAC:** Для нахождения геометрического преобразования между кадрами используется матрица гомографии $H$. Чтобы исключить влияние ложных сопоставлений (аутлайеров), применяется алгоритм RANSAC.

### 3. Описание алгоритма
Алгоритм работает по принципу конечного автомата с двумя состояниями: `FLOW` (основной режим) и `RECOVERY` (режим восстановления).

#### Основные этапы:
1.  **Инициализация:**
    * Пользователь вручную выделяет объект (ROI).
    * Внутри выделенной области находятся стабильные точки с помощью `cv2.goodFeaturesToTrack` (детектор Ши-Томаси).
    * Вычисляются эталонные SIFT-дескрипторы объекта для будущего восстановления.
2.  **Режим FLOW (Оптический поток):**
    * Точки с предыдущего кадра проецируются на текущий через `calcOpticalFlowPyrLK`.
    * На основе перемещения точек вычисляется матрица гомографии, которая трансформирует рамку (bounding box).
    * Если количество успешно отслеженных точек падает ниже порога (10), система переходит в режим `RECOVERY`.
3.  **Режим RECOVERY (Восстановление):**
    * На каждом новом кадре ищутся SIFT-признаки.
    * Дескрипторы текущего кадра сопоставляются с эталонными через `FlannBasedMatcher`.
    * При нахождении достаточного количества совпадений (>15) строится гомография, восстанавливается положение рамки, и алгоритм возвращается в режим `FLOW`.
4.  **Визуализация:**
    * Объект обводится зеленым контуром.
    * Отрисовываются текущие активные точки трекинга.
    * Выводится текстовая метка "Mona Lisa" (или "Recovering..." в режиме поиска).

### 4. Обсуждение результатов
Реализованный гибридный подход демонстрирует высокую устойчивость. 
* **Оптический поток** обеспечивает плавность и низкую вычислительную нагрузку при стабильном движении. 
* **SIFT-восстановление** позволяет системе «прощать» резкие рывки камеры или кратковременное перекрытие объекта, когда точки Lucas-Kanade выходят за пределы окна поиска.

При изменении ракурса рамка объекта корректно трансформируется в произвольный четырехугольник благодаря использованию `cv2.perspectiveTransform`, что позволяет визуально оценить перспективные искажения.

### 5. Выводы
В ходе работы был разработан продвинутый инструмент трекинга. Использование двух режимов работы решило главную проблему простых трекеров — окончательную потерю объекта при выходе за границы кадра или резком изменении освещения. 

**Ключевые преимущества:**
* Устойчивость к потере цели.
* Точность позиционирования за счет комбинации локальных признаков и глобального поиска.
* Инвариантность к поворотам и масштабированию.
